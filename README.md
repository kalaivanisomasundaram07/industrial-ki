## 🏗️ System Architecture

IKI implements a **Graph-Based Retrieval-Augmented Generation (GraphRAG)** pattern, orchestrated via a LangGraph multi-agent pipeline. Every claim generated by the Large Language Model is validated against a structured Knowledge Graph and raw text passages before being displayed to the user.

```mermaid
flowchart TD
    subgraph Client [Frontend UI Client]
        UI[Interactive Dashboard]
        Viewer[Two-Panel PDF Viewer]
    end

    subgraph API [FastAPI Backend Engine]
        Endpoint[/query Endpoint]
        Ingest[/ingest Endpoint]
    end

    subgraph LangGraph [Multi-Agent LangGraph Pipeline]
        Orch[Orchestrator Node]
        Router[Retrieval Router Node]
        RCA[RCA Agent Node]
        Comp[Compliance Agent Node]
        Gen[Answer Generation Node]
        Verify[Citation Verifier Node]
    end

    subgraph Storage [Heterogeneous Database Layer]
        SQLite[(SQLite DB: Meta & History)]
        Neo4j[(Neo4j: Knowledge Graph)]
        Chroma[(ChromaDB: Vector Store)]
    end

    UI -->|JSON Query| Endpoint
    Endpoint -->|Invoke State| Orch
    
    Orch -->|Route Query| Router
    Router -->|Hybrid Retrieval| Storage
    Router -->|RCA Query| RCA
    Router -->|General Query| Gen
    Orch -->|Compliance Query| Comp
    
    RCA --> Gen
    Comp --> Verify
    Gen --> Verify
    
    Verify -->|Fact-Check & Grounding| Storage
    Verify -->|Audit Results| Endpoint
    Endpoint -->|Response + Citations + Telemetry| UI
    
    Ingest -->|OCR / Text Parsing| SQLite
    Ingest -->|Triple Extraction & Review Queue| Neo4j
    Ingest -->|Sentence Embeddings| Chroma
```

## 🌟 Key Core Features

### 🤖 Multi-Agent LangGraph Orchestration
Queries enter a structured state machine with the following agents:
* **Orchestrator Agent**: Inspects the user's intent and routes traffic dynamically to domain-specific nodes based on query signatures (RCA, Compliance, or General).
* **Retrieval Router Agent**: Handles hybrid context retrieval, combining vector queries with sub-graph extractions.
* **RCA (Root Cause Analysis) Agent**: Traverses causality links (`CAUSED_BY`, `EXHIBITED_SYMPTOM`) to trace systemic operational issues.
* **Compliance Agent**: Direct-queries the regulatory framework mapping in Neo4j, bypassing LLM drafting to report verified standards.
* **Answer Generation Agent**: Synthesizes retrieved context using Groq Llama-3.3-70B to formulate clear, concise, and context-bounded responses.
* **Citation Verifier Agent**: Computes semantic grounding scores and flags logical contradictions.

### 🧬 Hybrid GraphRAG (Neo4j + ChromaDB)
* **Dense Vector Search**: Embeds chunks using SentenceTransformers (`all-MiniLM-L6-v2`) and indexes them in ChromaDB.
* **Structured Subgraphs**: Performs 2-hop Cypher queries around extracted query entities in Neo4j to pull structural node/relationship context.

### 🛡️ Grounding & Claim Verification Layer
* **Reranking Validation**: Uses a Cross-Encoder (`cross-encoder/ms-marco-MiniLM-L-6-v2`) to compare LLM sentences against document text, calculating a grounding similarity score.
* **Conflict Checking**: Automatically categorizes facts into:
  * **Verified**: Found directly in the Knowledge Graph.
  * **Conflict**: Claims that directly contradict a graph scalar relationship (e.g. pressure/temperature limits or equipment status).
  * **New Info**: Unverified claims not yet committed to the graph.

### 📥 Human-in-the-Loop Ingestion Queue
* Parses PDFs, Word Documents, Excel Spreadsheets, and images (with OCR support via `pytesseract`).
* Computes document credibility weights.
* Extracts RDF triples using the LLM and queues them in a `review_queue` table. Triples are committed to the Neo4j graph only after admin approval.

### 📊 Advanced Domain Engines
* **Risk Engine**: Computes equipment-level safety indexes and generates warning recommendations using failure-path graph structures.
* **Compliance Engine**: Performs audits comparing Standard Operating Procedures (SOPs) against regulatory targets, building gap logs and compiling markdown audit reports.
* **Pattern Engine**: Scans Cypher structures to highlight warning signs, operational anomalies, and critical warning cards.

## 🗄️ Database Schema & Data Model

### SQLite Local Storage (`industrial_ki.db`)
Stores system state, logs, and metadata across four primary tables:
1. `documents`: Ingested document registry, file paths, extensions, and ingestion logs.
2. `review_queue`: Gated extraction triples (`triple_json`, `evidence_snippet`, `score`, `status`).
3. `performance_stats`: Sub-millisecond timing records (graph query times, vector store search times, LLM generation, grounding checks).
4. `chat_history`: Conversation thread history (`role`, `content`, `metadata`, `session_id`, `created_at`).

### Neo4j Knowledge Graph Model
```
(Equipment) -[:HAS_COMPONENT]-> (Component)
(Component) -[:EXHIBITED_SYMPTOM]-> (Symptom)
(Symptom) -[:CAUSED_BY]-> (Failure)
(Failure) -[:RESOLVED_BY]-> (Action)
(Procedure) -[:GOVERNED_BY]-> (Regulation)
(Engineer) -[:PERFORMED_BY]-> (Action)
(Document) -[:DOCUMENTED_IN]-> (Equipment/Procedure/Regulation)
```

## ⚙️ Installation & Setup

### Prerequisites
* **Python 3.10+**
* **Node.js 18+**
* **Neo4j Desktop** (or a Neo4j Aura Instance)
* **Tesseract OCR** (for image text parsing)

### Step 1: Clone and Configure Environment
Copy the configuration template:
```bash
cp .env.example .env
```
Configure `.env` with your API keys and credentials:
```env
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_secure_password
GROQ_API_KEY=gsk_your_groq_api_key
CHROMA_PERSIST_DIR=./chroma_db
DATA_DIR=./data/raw
SQLITE_PATH=./industrial_ki.db
```

### Step 2: Install Backend Dependencies
```bash
cd backend
pip install -r requirements.txt
```

### Step 3: Run Database Migrations
Run schema constraints setup for Neo4j:
```bash
python graph/schema.py
```

### Step 4: Install Frontend Dependencies
```bash
cd ../frontend
npm install
```

### Step 5: Start the Application
Run both servers concurrently using the provided start script:
```bash
./start.sh
```
Or start them individually:
* **Backend**: `uvicorn backend.main:app --reload --port 8000`
* **Frontend**: `npm run dev` (running on `http://localhost:5173`)

## 🔌 API Reference

### Core Query Pipeline
#### `POST /query`
Executes the agentic reasoning pipeline.
* **Payload**:
  ```json
  {
    "question": "Why did Boiler-12 fail?",
    "session_id": "optional-session-uuid"
  }
  ```
* **Response**:
  ```json
  {
    "answer": "Boiler-12 experienced a steam leak due to a valve rupture...",
    "grounding_score": 0.89,
    "citations": [
      {
        "filename": "boiler_maintenance_log_2026.pdf",
        "page_number": 4,
        "similarity_score": 0.92
      }
    ],
    "conflicts": [],
    "causal_chain": [{"equipment": "Boiler-12", "failure": "rupture"}]
  }
  ```

### Chat History
* `GET /chat/sessions`: Lists all distinct chat sessions containing their first message, creation date, and message count.
* `GET /chat/history?session_id=X`: Returns the last 50 messages for the specified session.
* `DELETE /chat/history`: Wipes all logs from the database.

### Ingestion Queue
* `POST /ingest`: Scans raw files in `DATA_DIR`, extracts nodes/relationships, and populates the SQLite review queue.
* `GET /review-queue`: Fetches pending triples waiting for manual verification.
* `POST /review-queue/{id}/approve`: Approves and commits the triple directly to Neo4j.
* `POST /review-queue/{id}/reject`: Rejects the triple and removes it from the queue.

### Maintenance & Audit
* `GET /maintenance/risk`: Runs risk index heuristics and returns safety evaluations for all equipment.
* `GET /compliance/gaps`: Returns identified discrepancies between SOPs and regulatory guidelines.
* `POST /compliance/report`: Synthesizes a structured markdown compliance report using Llama-3.3.
* `GET /stats/performance`: Returns telemetry logs (average graph traversal times, RAG vector lookup times, and verifications).

## 💻 Frontend Dashboard Walkthrough

### 💬 Sidebar & Conversation History
* Left sidebar displays your previous chat threads grouped by session.
* Includes a **New Chat** button to instantly generate a clean session context.
* Support for a mobile overlay menu (under `768px`) toggled via a hamburger menu.

### 🔍 Document Viewer & Amber Highlighting
* Clicking on references inside chat triggers the Document Viewer.
* **Two-panel view**:
  * **Left (65%)**: Live PDF iframe pointing exactly to the cited page (`#page=N`).
  * **Right (35%)**: Shows the raw text chunk highlighted in amber, a search hint for easy navigation (using the first 5 words of the passage), and a clipboard copy helper.
